# UI Reference

The UI design language is defined in:

- [UI-DESIGN-README.md](C:\Users\rs329\goal\my-app\UI-DESIGN-README.md)

## Product Tone

The UI should feel like lead intelligence software.

It should communicate:

- signal detection
- focus
- credibility
- operational clarity

## Visual Direction

- dark-neutral foundation
- green signal accent
- compact, intentional cards
- high-contrast text
- structured layouts
- subtle gradients and grid textures

## Component System

Default UI foundation:

- Tailwind CSS
- `shadcn/ui`-style components

Preferred primitives:

- `Button`
- `Card`
- `Input`
- `Badge`
- `Dialog`
- `DropdownMenu`
- `Table`
- `Toast`

## Auth UI Direction

- split layout on desktop
- product story on the left
- auth card on the right
- toast-based error handling
- Google sign-in primary action

## Screen Principles

### Campaigns

Should feel structured and configurable.

### Leads

Should feel like an intelligence queue.

### Lead Detail

Should feel analytical and context-rich.

### Settings

Should feel operational and simple.

## Avoid

- flat white SaaS styling
- generic purple branding
- excessive blur effects
- overly playful visuals
- mixing different UI systems on the same page
